# ドキュメント管理ガイドライン（指示書）

AIが新しいプロジェクトドキュメント（執筆物、レポート、計画書）を作成・整理する際は、以下のルールを厳守してください。

## 1. 基本方針
- **ルートディレクトリの保護**: 実行スクリプトや設定ファイル、現在進行中のコンテンツ以外の「読み物」は、原則ルートに置かず `Documents/` フォルダへ格納する。
- **依存関係の優先**: コードが直接参照しているMDファイルや、プロジェクトのルートにあるべき `README.md` などは移動しない。

## 2. フォルダ構造と格納基準
作成するファイルの内容に応じて、以下のサブフォルダに分類して格納してください。

| フォルダ名 | 格納すべき内容の例 |
| :--- | :--- |
| `Documents/Reports/` | 実装完了レポート、分析結果、構成提案、定例活動記録（Activity Log） |
| `Documents/Plans/` | 今後の開発計画、スキルのアップグレード案、未実施のロードマップ |
| `Documents/Guides/` | ツール・スキルの使い方の解説、ワークフローの定義、ルールブック |

## 3. ファイル命名規則
- **日付を付与**: レポートやログは、後から時系列で追えるよう `YYYY-MM-DD_ファイル名.md` の形式を使用する。
- **内容が推測できる名前**: `Report.md` などの曖昧な名前は避け、`2026-01-28_Video_Gen_Implementation_Report.md` のように具体的に記述する。

## 4. AIへの具体的な指示の出し方（例）
次のような指示をいただければ、このガイドラインに沿って動きます。
> 「今回の作業内容をレポートにまとめて。保存先は例のドキュメント管理ルールに従って `Documents` 内に適切に分けておいて。」
