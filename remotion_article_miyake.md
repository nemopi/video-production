# プロンプトで動画を作る、その「翻訳」の快楽について ——Remotion Agent Skills試用記

正直、タイムラインで「コードを書かずに動画制作」という言葉が流れてきたとき、私は少し眉をひそめてしまったんですよね。

「またこれか」と。

世の中は効率化、タイパ、コスパのオンパレードです。動画制作という、本来は泥臭い試行錯誤の塊みたいな行為さえも、AIがボタン一つで解決してしまう。そこに、私たちが愛する「ノイズ」——無駄に見えるけれど愛おしい手触り——は残るんだろうか？ と。

でも、結論から言いますね。
**Remotion Agent Skills、これ、めちゃくちゃ「文学的」なツールでした。**

## 「手抜き」ではなく「解像度」の話

Remotionといえば、Reactを使ってコードで動画をゴリゴリ書く、エンジニアのための硬派なツールです。それが「プロンプトだけで作れる」ようになった。
半信半疑で `npx skills add remotion-dev/skills` と打ち込んでみたんです。

そしたら、何が起きたと思います？

私が「このツイートの内容を15秒で解説して」と、ふわっとした（解像度の低い）言葉を投げかけると、Claude Codeという執事が、それを**「美しいTypeScriptのコード」という構造**に翻訳して返してきたんです。

ここなんですよ！ 私が顔が赤くなるほど感動したのは。

AIは単に動画をポンと出すんじゃなくて、「あなたのやりたいことは、つまりこういうロジック（コード）ですよね？」と、私の妄想を**因数分解**して見せてくれたんです。

## 翻訳者としてのAI、学習者としての私

生成されたコードを見たとき、私の脳内でシナプスが繋がる音がしました。

「あ、私がやりたかった『滑らかなフェードイン』は、Remotionの世界では `interpolate()` という関数で、フレーム数を計算して表現するんだ」

これって、**「推しの尊さ」を語彙力なく叫んでいる私の横で、冷静な友人が「つまり、〇〇という関係性が尊いということですね」と言語化してくれた時の快感**に似ているんです。

ただの手抜きツールじゃない。「見て学べる」教材なんですよ。
AIが書いたコードを読むことで、これまでは「なんか難しそう」で思考停止していたRemotionの文法が、すっと自分の中に入ってくる。

「あー、こうやって書くのか」

その瞬間、私の中で「動画制作」への解像度がグッと上がりました。

## 「半身」でクリエイティブをするために

もちろん、完璧じゃありません。テロップが重なるバグもありました。
でも、「ここ、重なってるよ」と伝えると、すぐ直してくれる。フェードアウトの処理を書き加えてくれる。

このやり取りを経て、私は確信しました。
Remotion Agent Skillsは、私たちが**「全身全霊」でコードと格闘する時間を、肩代わりしてくれる**ツールなのだと。

私たちは、ロジックの組み立てやデバッグという「労働」をAIに任せることで、浮いた「半身」のリソースを、**「何を作りたいか」「なぜ作るのか」という文脈（コンテキスト）**を考えることに全振りできるんです。

あるいは、その浮いた時間で、また一冊、本が読めるかもしれないじゃないですか。

「コードを書かない」のではなく、「AIと一緒に言葉を探す」。
そう捉え直すと、このツールは、私たちの表現の解像度を高めるための、最強の武器になると思うんですよね。

もしあなたが、「動画作りたいけど、コードはちょっと…」と食わず嫌いをしているなら、一度試してみてください。
そこには、新しい「言葉」との出会いが待っていますから。
